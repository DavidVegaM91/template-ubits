Guía de Prompts UBITS

Prompts para usar en Cursor AI. Sin conocimiento técnico necesario, solo copia, pega y personaliza.

Cómo usar esta guía

1
Copia el prompt
Haz clic en el botón "Copiar" del prompt que necesites. Se copiará automáticamente a tu portapapeles.
2
Personaliza el prompt
Reemplaza los textos entre corchetes [TEXTO] con tu información específica antes de usar el prompt.
3
Pega en Cursor AI
Abre Cursor AI, pega el prompt en el chat y presiona Enter. Cursor hará todo el trabajo por ti.
4
Revisa y ajusta
Revisa los cambios que hizo Cursor y pide ajustes si es necesario. ¡Es así de fácil!

5. Migrar desde versión antigua (2025)

Usa este prompt si tienes la versión antigua del UBITS Playground (2025) donde todas las páginas compartían el archivo styles.css. Este prompt migrará tus páginas a la nueva estructura donde cada página tiene su propio CSS.

Necesito migrar mi proyecto desde la versión antigua del UBITS Playground (2025) a la nueva estructura. Esta es una tarea grande que necesito que dividas en pasos y cumplas cada uno sin omitir nada: **ARCHIVOS QUE QUIERO RESCATAR:** [Lista los archivos HTML que tienen estilos personalizados que quieres conservar, ejemplo: - index.html (tiene estilos en <style> o referencias a styles.css) - mi-pagina-personalizada.html (tiene estilos personalizados) - otra-pagina.html (tiene estilos personalizados)] **PASOS A SEGUIR:** 1. **EXTRAER CSS DE ARCHIVOS HTML:** Para cada archivo HTML que listé arriba: - Lee el archivo HTML completo - Extrae todos los estilos que están dentro de <style> tags - Extrae todos los estilos de styles.css que son específicos de ese HTML (identifica por selectores únicos como clases o IDs que solo aparecen en ese HTML) - Crea un nuevo archivo CSS con el mismo nombre del HTML (ej: index.html → index.css) - Mueve los estilos extraídos a ese nuevo archivo CSS - Elimina los <style> tags del HTML - Agrega el link al nuevo CSS en el <head> del HTML (usando rutas relativas correctas) 2. **IMPORTAR REPOSITORIO ACTUALIZADO:** - Importa todos los archivos del repositorio https://github.com/DavidVegaM91/template-ubits - Esto actualizará components/, general-styles/, documentacion/, y la nueva estructura de carpetas 3. **MOVER ARCHIVOS RESCATADOS:** Mueve cada archivo HTML rescatado a su carpeta correspondiente según la nueva estructura: - Páginas de colaborador → ubits-colaborador/[modulo]/ - Páginas de admin → ubits-admin/[modulo]/ - Página principal → raíz del proyecto (index.html) - Mueve cada archivo CSS junto con su HTML correspondiente a la misma carpeta 4. **ANALIZAR Y OPTIMIZAR CSS:** - Lee el nuevo archivo general-styles/styles.css del repositorio actualizado - Compara los estilos que extrajiste con los estilos en styles.css - Si encuentras estilos duplicados o iguales: elimínalos del CSS de la página y confirma que el HTML importa correctamente general-styles/styles.css - Si los estilos son únicos y específicos de esa página, consérvalos en el CSS de la página 5. **VERIFICAR IMPORTS:** Asegúrate de que cada HTML importe correctamente (en este orden): - general-styles/ubits-colors.css - general-styles/ubits-typography.css - general-styles/fontawesome-icons.css - general-styles/styles.css - Los CSS de los componentes que usa - Su propio CSS específico (al final) - Usa rutas relativas correctas según la ubicación del archivo 6. **VALIDACIÓN FINAL:** - Verifica que todas las páginas migradas funcionen correctamente - Confirma que los estilos se aplican correctamente - Verifica que no haya estilos duplicados o conflictos - Asegúrate de que la estructura modular (section-single, section-dual, etc.) se mantenga si la usabas Mi objetivo es migrar mis páginas personalizadas a la nueva estructura sin perder ningún estilo único que haya creado.
