Progreso segmentado

Muestra avance ítem a ítem en quiz, flashcards o flujos con pasos discretos. Con pocos ítems usa segmentos; con muchos (20+) cambia a modo slider con indicador.

Preview del componente

Usa los controles para cambiar el ítem activo. Con 20 o más ítems el modo pasa a slider automáticamente.

Características generales

Dos modos
Segmentos (≤19 ítems) o slider con thumb (20+).
Dos alturas
sm (6px) y lg (8px en slider).
HTML o función
segmentedProgressHtml o createSegmentedProgress; actualiza con setSegmentedProgressValue.

Prompts útiles

Insertar progreso segmentado

Inserta un Progreso segmentado UBITS en [ubicación] con [número] ítems y el ítem activo en [posición]. Importa segmented-progress.css y segmented-progress.js.

Actualizar ítem activo

Cuando el usuario avance al ítem [n] en [flujo], actualiza el progreso segmentado con setSegmentedProgressValue sobre el nodo .ubits-segmented-progress.

Documentación técnica

Estructura HTML, API y requisitos de importación.

Requisitos

  • components/segmented-progress.css
  • components/segmented-progress.js
  • Tokens UBITS vía styles.css

segmentedProgressHtml

segmentedProgressHtml({
  total: 10,           // número de ítems
  current: 0,          // índice activo (0-based)
  mode: 'auto',        // auto | segments | slider
  sliderThreshold: 19, // auto → slider si total > 19
  size: 'sm',          // sm | lg
  track: 'default',    // default | subtle (bg más claro, flashcards)
  id: 'mi-progreso',
  ariaLabel: 'Progreso del quiz'
});

setSegmentedProgressValue(document.getElementById('mi-progreso'), 3);
