Carousel indicators

Indicadores al estilo Swiper “dynamic bullets”: como máximo seis puntos visibles aunque el carrusel tenga muchas diapositivas; la ventana se desplaza para centrar el activo. Con pocas diapositivas se muestran todas. Con 11 o más slides el modo dinámico mantiene seis puntos y menos espacio entre ellos. El activo es una pastilla horizontal en color de marca; los inactivos son círculos que se achican y pierden opacidad hacia los bordes, con transición suave al cambiar de slide. Sin caja ni borde.

Preview del componente

Prueba con 2 a 6 o 28 diapositivas: con más de seis slides solo ves seis puntos (ventana deslizante); a partir de 11 entra el modo dinámico (clase --dynamic, estilo Swiper). El avance se nota en el morph pastilla ↔ puntos y en la opacidad de los extremos.

Características generales

Hasta seis puntos a la vez
La ventana muestra como máximo seis indicadores. Con 11 o más diapositivas entra el modo dinámico (ventana deslizante de índices). Cada punto lleva a una slide real.
Transición al cambiar
Al cambiar de slide, la pastilla activa y los puntos vecinos morphan (ancho, alto, color, opacidad) con curva fluida, como en paginación dinámica de Swiper.
Solo la fila de puntos
Sin contenedor con borde ni fondo propio: únicamente los indicadores, listos para ponerlos encima del carrusel o del color que elijas.

Prompts útiles

Añadir indicadores de carrusel

Inserta el componente Carousel indicators debajo de [ubicación] para un carrusel con [número] diapositivas. Importa carousel-indicators.css y carousel-indicators.js. Al cambiar de slide, actualiza el indicador con la API del componente.

Sincronizar con tu carrusel

Cuando el usuario cambie de diapositiva en [nombre del carrusel], llama a setActive en el objeto devuelto por initCarouselIndicators para mantener los puntos alineados con la vista.

Estructura, API y requisitos para desarrolladores.

Requisitos

  • general-styles/ubits-colors.css y styles.css (espaciado)
  • components/carousel-indicators.css

maxVisible (1–6, default 6), dynamicFrom (default 11): si count ≥ dynamicFrom, la ventana efectiva es de 6 puntos; dynamicFrom: false desactiva ese modo. Los mismos button se reutilizan: el cambio de slide es morph CSS cuando el activo cambia de botón. Si el mismo botón sigue activo (ventana centrada), un fotograma dist-2 y luego de nuevo active (dos requestAnimationFrame) vuelve a disparar el morph pastilla↔punto sin translate. Inspiración UX: Swiper pagination · dynamicBullets.

initCarouselIndicators(options)

initCarouselIndicators({
  containerId: 'mi-carousel-dots', // o container: elemento HTMLElement
  count: 15,
  activeIndex: 0,
  maxVisible: 6,
  dynamicFrom: 11,
  ariaLabel: 'Indicadores de diapositiva',
  onSelect: function (index) { /* ir a slide index */ }
});
// Devuelve: { setActive(i), setCount(n), getState(), destroy() }
// getState(): count, activeIndex, maxVisible, dynamicFrom, effectiveVisible
