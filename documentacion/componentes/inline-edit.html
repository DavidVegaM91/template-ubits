Inline Edit

Campo de texto que se puede editar directamente en la pantalla, sin abrir un formulario aparte. Sirve para títulos, descripciones y cualquier texto que el usuario deba poder cambiar en el momento, sin interrumpir el flujo.

Preview del componente

Haz clic dentro del campo para editarlo. Prueba las diferentes variantes tipográficas con los controles de abajo.

Características generales

Hereda la tipografía
Funciona con cualquier clase tipográfica UBITS: display, heading o body.
Placeholder diferenciado
El texto guía usa el mismo color que los inputs UBITS, claro en claro y claro en oscuro.
Auto-resize automático
El campo crece con el contenido: nunca aparece barra de scroll ni texto cortado.
Fondo sutil al interactuar
Al pasar el cursor o hacer foco aparece un fondo suave que indica que el campo es editable.
Modo oscuro incluido
Todos los colores usan tokens UBITS: cambian solos al cambiar de tema.
Variante de solo lectura
Atributo readonly desactiva la edición sin cambiar la apariencia del texto.

En [nombre de la función de renderizado de la página], después de insertar el HTML en el DOM, llama a initInlineEdit(contenedor) para activar el auto-resize de todos los textarea.ubits-inline-edit dentro de ese contenedor.

Estructura del componente

Siempre combina ubits-inline-edit con una clase tipográfica UBITS. Para múltiples líneas usa <textarea>; para una sola línea, <input type="text">. 

<!-- Título grande (multilinea, auto-resize) -->
<textarea
    class="ubits-inline-edit ubits-display-d3-bold"
    placeholder="Escribe el título del contenido..."
    rows="1"
></textarea>

<!-- Encabezado de sección (multilinea) -->
<textarea
    class="ubits-inline-edit ubits-heading-h2"
    placeholder="Nombre de la sección..."
    rows="1"
></textarea>

<!-- Descripción (multilinea) -->
<textarea
    class="ubits-inline-edit ubits-body-md-regular"
    placeholder="Añade una descripción..."
    rows="1"
></textarea>

<!-- Una sola línea fija (sin auto-resize, no necesita JS) -->
<input
    type="text"
    class="ubits-inline-edit ubits-heading-h1"
    placeholder="Nombre del plan..."
>

<!-- Solo lectura -->
<textarea
    class="ubits-inline-edit ubits-body-md-regular"
    readonly
>Texto no editable</textarea>

Cómo implementar 

<!-- 1. Importar CSS (después de ubits-typography.css) -->
<link rel="stylesheet" href="../../components/inline-edit.css">

<!-- 2. Importar JS (solo si usas textarea; input no lo necesita) -->
<script src="../../components/inline-edit.js"></script>

<!-- 3. HTML -->
<textarea
    id="mi-titulo"
    class="ubits-inline-edit ubits-display-d3-bold"
    placeholder="Escribe el título..."
    rows="1"
></textarea>

<!-- 4. Inicializar (el JS se auto-inicia al cargar la página,
        pero si insertas HTML dinámicamente, llama a: -->
<script>
    // Tras renderizar HTML con inline-edits:
    initInlineEdit(document.getElementById('mi-contenedor'));

    // Si cambias el valor por código:
    const campo = document.getElementById('mi-titulo');
    campo.value = 'Nuevo título';
    autoResizeInlineEdit(campo);
</script>

API y funciones

initInlineEdit(root?)

Inicializa el auto-resize en todos los textarea.ubits-inline-edit dentro de root. Si no se pasa root, busca en todo el documento. Usa un guard interno para evitar doble binding.

Parámetro: root — HTMLElement o Document (opcional, default: document)
Ejemplo: initInlineEdit(document.getElementById('mi-sección'))

autoResizeInlineEdit(el)

Ajusta la altura de un textarea individual al contenido actual. Úsalo cuando cambies el valor del campo por código (campo.value = '...').

Parámetro: el — HTMLTextAreaElement
Ejemplo: autoResizeInlineEdit(document.getElementById('mi-titulo'))

Clases tipográficas compatibles

Cualquier clase tipográfica UBITS funciona como companion de ubits-inline-edit. El color del texto lo fija la clase tipográfica: display y heading → fg-1-high, body → fg-1-medium.
