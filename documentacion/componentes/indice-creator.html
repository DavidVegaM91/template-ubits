Índice Creator

Panel izquierdo del paso Recursos en la creación de contenidos: interruptor para usar secciones (módulos), listado de secciones y páginas, y «Añadir sección» al pie cuando las secciones están activas. Si las secciones están desactivadas, todas las páginas se muestran en un solo bloque sin cabecera de sección. Referencia Figma: Learn-Components · Índice (242:5621). Contexto de producto: contexto-creacion-contenido.md.

Preview del componente

Datos de ejemplo alineados con Figma (242:5621): curso de ejemplo sobre gestión de conflictos. El interruptor solo cambia la demo entre lista única (sin cabeceras de sección) y varias secciones + «Añadir sección». Clic en una página: queda activa solo esa fila y su sección (borde de marca); el resto de secciones y páginas se desactivan. En producto conviene persistir el estado al cambiar el interruptor interno.

Características generales

Interruptor de secciones
Por defecto desactivado en producto; al activarlo aparecen varias secciones y el botón para añadir más.
Contenidos largos
Las secciones ayudan a organizar cursos extensos; en piezas cortas suele bastar una sola lista de páginas.
Añadir sección
Solo visible cuando el uso de secciones está activo; secundario a ancho completo bajo el índice.
Una sola lista
Sin secciones, no se muestra cabecera de sección: solo páginas y «Añadir página» en el mismo bloque.

Prompts útiles

Copia y personaliza. Pide importar switch.css, paginas-creator.css, seccion-creator.css, indice-creator.css, button.css, dropdown-menu.css y los JS en orden: dropdown-menu.js, tooltip.js, paginas-creator.js, seccion-creator.js, indice-creator.js.

Índice del creador en el paso Recursos

En el panel izquierdo del paso Recursos en [ubicación], coloca el componente UBITS «Índice Creator»: interruptor «Habilitar el uso de secciones», y debajo el listado (secciones con páginas y «Añadir página», o una sola lista de páginas sin título de sección si las secciones están desactivadas). Si las secciones están activas, añade al final el botón «Añadir sección». Importa los CSS y JS del componente para que no se vea roto.

Estructura y API

Contenedor: .ubits-indice-creator
Tarjeta interruptor: .ubits-indice-creator__toggle-card (label + .ubits-switch)
Cuerpo: .ubits-indice-creator__body
  · Con secciones: .ubits-indice-creator__index-wrap > .ubits-seccion-creator-index … + .ubits-indice-creator__add-section > botón .ubits-indice-creator__add-section-btn
  · Sin secciones: .ubits-indice-creator__single-wrap > .ubits-seccion-creator__section--no-header (sin cabecera; solo páginas + «Añadir página»)

indiceCreatorHtml({
  sectionsEnabled: true|false,  // false por defecto en producto; true = varias secciones + «Añadir sección»
  toggleId?, toggleLabel?,
  sections?, indexOpts?,        // modo secciones → seccionCreatorIndexHtml
  pages?, singleSection?, singleSectionKey?, addSectionButtonId?, addPageLabel?
})

initIndiceCreator(rootSelector | elemento) — delega initSeccionCreator; cablea toggle y «Añadir sección»

Eventos:
  ubits-indice-creator-sections-toggle — detail: { sectionsEnabled, root }
  ubits-indice-creator-add-section — detail: { root }
(Reutiliza ubits-seccion-creator-* y ubits-paginas-creator-* dentro del bloque)
