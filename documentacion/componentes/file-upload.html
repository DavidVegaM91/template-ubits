File Upload

Bloque para importar archivos con arrastre o selección manual. Incluye descarga de plantillas, validación automática de tipo y peso, card con el archivo cargado y botón opcional de informe de errores cuando el contenido presenta inconsistencias.

Preview del componente

Aquí puedes ver el componente File Upload en acción. Usa los controles para explorar los distintos estados.

Características generales

Drag & drop
Arrastra el archivo sobre la zona o haz clic para seleccionarlo desde el explorador.
Validación automática
Valida tipo de archivo y tamaño al seleccionarlo, mostrando un mensaje de error claro.
Botones de descarga
Hasta 3 botones en el encabezado para descargar plantillas o listas de referencia.
Informe de errores
Botón error-secondary oculto por defecto; se activa cuando el servidor detecta problemas en el contenido.
Card de archivo
Muestra nombre y tamaño del archivo cargado con botón para eliminarlo y volver al estado inicial.
Modo oscuro
Todos los colores usan tokens UBITS, compatible con tema claro y oscuro automáticamente.

Prompts útiles

Usa estos prompts con Cursor AI para insertar y configurar el componente File Upload.

Insertar bloque de importación CSV 

Añade el componente ubits-file-upload en [ubicación] de la página. Configúralo con:
- Título: "Importar archivo"
- 1 botón de descarga: "Descargar plantilla" (icono arrow-down-to-line)
- Tipos aceptados: .csv
- Límite de peso: 5 MB
Importa components/file-upload.css antes del CSS de la página y components/file-upload.js antes de cerrar el body. Reemplaza los textos entre [corchetes].

Mostrar botón de informe de errores 

Cuando el servidor devuelva errores al procesar el archivo en [ubicación], llama a fileUploadShowErrorReport('[id-del-componente]', true) para mostrar el botón "Informe de errores". Ese botón debe descargar el archivo de errores que devuelve el servidor.

Importación de SCORM con múltiples descargas 

Añade el componente ubits-file-upload en [ubicación] para subir paquetes SCORM. Configúralo con:
- Título: "Importar SCORM"
- 2 botones de descarga: "Requisitos técnicos" y "Lista de compatibilidad"
- Tipos aceptados: .zip
- Límite de peso: 100 MB
Importa components/file-upload.css y components/file-upload.js.

Mostrar un error externo (del servidor) 

Cuando la API devuelva un error al procesar el archivo en [ubicación], llama a fileUploadSetError('[id-del-componente]', '[mensaje de error]') para mostrar el borde rojo y el mensaje debajo del dropzone. Cuando el usuario quite el archivo o corrija la situación, llama a fileUploadClearError('[id-del-componente]').

Implementación con createFileUpload() 

createFileUpload({
  containerId: 'mi-contenedor',   // ID del div donde se inyecta (requerido)
  id: 'mi-file-upload',           // ID del elemento raíz generado (opcional)
  title: 'Importar archivo',      // Título del encabezado
  accept: '.csv,text/csv',        // Tipos aceptados (atributo accept)
  maxSizeMb: 5,                   // Límite de peso en MB
  formats: 'CSV \u2022 Hasta 5mb',    // Texto de formatos (opcional; derivado de accept si se omite)
  downloadButtons: [              // Hasta 3 botones de descarga
    {
      label: 'Descargar plantilla',
      icon: 'arrow-down-to-line',   // Clase FA sin el prefijo 'far fa-'
      onClick: function () { /* descargar plantilla */ }
    }
  ],
  onChange: function (file) {
    // file = File | null
    if (file) console.log('Archivo seleccionado:', file.name);
    else console.log('Archivo eliminado');
  },
  onError: function (err) {
    // err = { type: 'type' | 'size', message: string }
    console.warn(err.type, err.message);
  }
});

// ── API auxiliar ──────────────────────────────────────────────

// Mostrar / ocultar el botón "Informe de errores" (tras procesar en servidor)
fileUploadShowErrorReport('mi-file-upload', true);

// Mostrar error externo (respuesta del servidor)
fileUploadSetError('mi-file-upload', 'Se encontraron 3 usuarios que no existen.');

// Limpiar error inline
fileUploadClearError('mi-file-upload');
