Learn question

Bloque para crear y mostrar preguntas en flujos de aprendizaje y evaluaciones, con estados de edición, lectura y validación.

Preview del componente

Usa los controles para cambiar el tipo de pregunta y el modo (edición/lectura/error/colaborador).

Controles

El preview usa datos demo. En producción, el modelo viene del flujo (Creator/Modo estudio IA).

Componente

Características generales

Edición y lectura
Una pregunta puede estar activa (edición) o en lectura mientras editas otra.
Estados de error
Soporta lectura error y edición error para guiar al usuario a completar campos requeridos.
Modo colaborador
Permite responder sin mostrar cuáles eran correctas. Incluye una variante con feedback inmediato.

Prompts útiles

Insertar una pregunta (edición)

Inserta el componente Learn question en [ubicación] en modo Edición para una pregunta de tipo [tipo]. Importa `../../components/learn-question.css`, `../../components/learn-question.js`, `../../components/input.css`, `../../components/input.js`, `../../components/radio-button.css` y `../../components/checkbox.css`.

Mostrar preguntas al colaborador

En [pantalla], renderiza las preguntas usando el componente Learn question en modo Colaborador. No muestres cuáles eran las respuestas correctas. Importa `../../components/learn-question.css` y `../../components/learn-question.js` (y `input.css/input.js` si hay respuesta de texto).

Cómo implementar 

<div id="my-question"></div>
<script>
  const api = createLearnQuestion({
    containerId: 'my-question',
    qId: 1,
    mode: 'edit', // 'read' | 'read_error' | 'edit_error' | 'collab' | 'collab_feedback'
    model: {
      type: 'multiple_choice_single',
      statement: '¿Cuál es la respuesta correcta?',
      options: [
        { text: 'Opción A', correct: false },
        { text: 'Opción B', correct: true }
      ]
    },
    onDelete: (qid) => console.log('delete', qid),
    onRequestFocus: (qid) => console.log('focus', qid)
  });
</script>

API y funciones

createLearnQuestion(options): monta una pregunta y devuelve un API.
api.setMode(mode): cambia variante/estado de la pregunta.
api.getModel(): devuelve el modelo actual (serializable).
api.validate(): valida y retorna { ok, missing }.
