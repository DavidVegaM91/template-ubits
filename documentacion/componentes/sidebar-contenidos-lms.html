Sidebar contenidos LMS

Misma pieza que el Sidebar de la documentación (.sidebar, .sidebar-main, .sidebar-body, botones .nav-button con icono y data-tooltip en modo estrecho), pero con fondo claro (--ubits-bg-1) para ir dentro del LMS Creator junto al lienzo, sin sustituir el menú global oscuro de la app. Hay dos variantes de flujo: Publicado LMS Creator (cinco pasos, con Resultados) y Publicado Antiguo LMS (sin Resultados; el paso de recursos se muestra como Módulos, mismo icono). En el pie, un control con solo icono al estar contraído (tooltip “Expandir”) ensancha el rail y muestra el texto de cada paso; en ese modo los tooltips de los pasos no se muestran.

Preview del componente

Pasa el cursor sobre cada icono para ver el tooltip (como en el Sidebar). Abajo, el botón solo muestra el icono de doble flecha al estar contraído (tooltip “Expandir”); al expandir ves el texto de cada paso y en el pie aparece Contraer. Puedes activar un paso haciendo clic en el icono o con los botones de abajo. El panel gris simula el área de edición.

Características generales

Replica el comportamiento del rail del Sidebar global; solo cambian colores y que no va fijo a la vista del escritorio de la app.

Misma estructura
aside.sidebar, sidebar-main, sidebar-body y nav-button como en sidebar.js.
Colores para bg-1
Modificador sidebar--contenidos-lms: superficie clara, hover y activo legibles sobre fondo principal.
Variantes de pasos
Publicado LMS Creator: cinco pasos (Resultados → Visibilidad). Publicado Antiguo LMS: cuatro pasos, sin Resultados; data-step="recursos" con etiqueta Módulos. Rail con data-tooltip o modo expandido.

Cuando el usuario avance al paso [nombre del paso] en el asistente de creación de contenido, actualiza el ítem activo del Sidebar contenidos LMS con setSidebarContenidosLmsActive(idContenedor, stepId), sin recargar toda la barra.

Documentación técnica

API, archivos y estructura generada.

Mostrar

API (JavaScript) 

// Contenedor vacío en el HTML:
// <div id="mi-sidebar-creator"></div>

loadSidebarContenidosLms('mi-sidebar-creator', {
  // variant (opcional): 'publicado-lms-creator' (default) | 'publicado-antiguo-lms'
  // Antigua: sin resultados; recursos se muestra como "Módulos" (id sigue siendo 'recursos')
  variant: 'publicado-lms-creator',
  activeStep: 'recursos', // según pasos de la variante elegida
  onSelect: function (stepId) {
    // Mostrar panel correspondiente, actualizar URL, etc.
  }
});

// Solo cambiar resaltado (p. ej. tras navegar por otra UI):
setSidebarContenidosLmsActive('mi-sidebar-creator', 'certificado');
// o pasando el nodo aside:
// setSidebarContenidosLmsActive(document.querySelector('.sidebar--contenidos-lms'), 'visibilidad');

Estructura HTML (referencia) 

<aside class="sidebar sidebar--contenidos-lms" data-variant="publicado-lms-creator" aria-label="Pasos de creación de contenido">
  <div class="sidebar-main">
    <div class="sidebar-body">
      <button type="button" class="nav-button active" data-step="resultados"
        data-tooltip="Resultados" aria-label="Resultados" aria-current="page">
        <i class="far fa-chart-mixed" aria-hidden="true"></i>
        <span class="sidebar-contenidos-lms__step-label ubits-body-sm-regular">Resultados</span>
      </button>
      <!-- …información, recursos, certificado, visibilidad (mismo patrón) -->
    </div>
  </div>
  <div class="sidebar-contenidos-lms__footer">
    <button type="button" class="nav-button sidebar-contenidos-lms__toggle">…</button>
  </div>
</aside>

Equivale al armado del Sidebar en sidebar.js, sin header con logo. El pie añade Expandir/Contraer. Usa loadSidebarContenidosLms para generarlo. Modo expandido: clase sidebar--contenidos-lms--expanded en el aside (la interacción del pie la gestiona el JS del componente).

Requisitos

CSS: general-styles/styles.css (clases .sidebar, .nav-button), components/sidebar-contenidos-lms.css (modificador sidebar--contenidos-lms); ubits-colors.css, fontawesome-icons.css.
JS: components/sidebar-contenidos-lms.js.
Opcional: tooltip.css + tooltip.js; el loader llama a initTooltip si existe.
