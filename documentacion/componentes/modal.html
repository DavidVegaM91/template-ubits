Modal

Ventana que se abre encima de la página para confirmar algo, rellenar un formulario o elegir una fecha. Se cierra con la X, los botones o tocando fuera. Para flujos de generación con IA hay una variante con número de tokens en la cabecera y fondo con orbes de color. La variante promo sirve para anunciar novedades: sin cabecera ni pie ni botón X, imagen y texto con botón con padding (tamaños solo XS y SM); se cierra con Escape, tocando fuera o el CTA del cuerpo.

Preview del componente

Haz clic en un botón para abrir el modal en el tamaño indicado. Cierra con el botón X, Cancelar, Aceptar o clic fuera del contenido.

Características generales

Overlay
Fondo oscuro; al tocar fuera se cierra
Partes
Título con botón cerrar, contenido y pie con botones
Uso
Confirmaciones, formularios cortos, selección de fecha
Tamaños
Varios anchos del modal: pequeño, mediano y grande
Variante IA
Cabecera con tokens restantes y fondo con orbes; misma API con openModal
Variante promo
Sin barra de título ni pie; imagen full bleed y bloque con texto y CTA. Solo tamaños XS y SM.

Cerrar modal al clic en overlay

Haz que el modal [id del overlay] se cierre cuando el usuario haga clic fuera del contenido (en el overlay). Usa onclick en el overlay verificando event.target === this, y event.stopPropagation() en el contenido.

Modal de eliminación con botón rojo (error)

Usa el componente modal UBITS para el modal de confirmación de eliminación [id del overlay]. Título: "Confirmar eliminación". Body: mensaje de advertencia. Footer: botón Cancelar (ubits-button--secondary) y botón Eliminar (ubits-button--error ubits-button--md). El contenedor del modal debe usar ubits-modal-content ubits-modal-content--xs (320px). Clases: ubits-modal-overlay, ubits-modal-header, ubits-modal-body, ubits-modal-footer. El botón rojo está documentado en documentacion/componentes/button.html como variante "error".

Modal promo (anuncio sin cabecera)

Abre un modal promo con openModal en [ubicación]: variant promo, tamaño [xs o sm], imagen y debajo el texto [título] y [descripción] con un botón primario [texto del CTA]. Sin pie de modal ni botón X (cierre: Escape, fuera o CTA). Importa modal.css y button.css. Usa promoAriaLabel con un texto corto para accesibilidad.

Cómo implementar

🔧 REQUISITOS OBLIGATORIOS:

CSS: components/modal.css, ubits-colors.css, ubits-typography.css, fontawesome-icons.css (botón cerrar), components/button.css (botones del header/footer). Para la variante IA, además components/badge-tag.css (badge de tokens) y, si usas tooltip en el badge, components/tooltip.css + components/tooltip.js (o inicializas el tooltip tú tras abrir). Los orbes usan variables --modo-ia-glow-orb-rgb-1/2/3 (p. ej. aprendizaje-ia-gradientes.css en el playground).
JavaScript: components/modal.js (recomendado). Proporciona openModal(), closeModal(), showModal(), hideModal(). Si no usas JS, mostrar/ocultar y lógica a cargo de tu página.
Renderizado: Recomendado: openModal({ overlayId, title, bodyHtml, footerHtml?, size?, closeOnOverlayClick?, onClose? }) para estructura siempre correcta. Alternativa: HTML directo con las mismas clases (ver ejemplos abajo).

Con JavaScript (recomendado)

Una sola fuente de verdad: el título y la estructura los genera openModal(), así no se desincronizan otras páginas.

openModal({
  overlayId: 'mi-modal',
  title: 'Título del modal',
  bodyHtml: '<p class="ubits-body-md-regular">Contenido.</p>',
  footerHtml: '<button class="ubits-button ubits-button--secondary ubits-button--md"><span>Cancelar</span></button><button class="ubits-button ubits-button--primary ubits-button--md"><span>Aceptar</span></button>',
  size: 'sm',
  closeOnOverlayClick: true,
  onClose: function() { }
});
closeModal('mi-modal');

Variante IA (variant: 'ia')

Añade clase ubits-modal-content--ia, cabecera ubits-modal-header--ia y, si pasas iaTokensRemaining, el badge de tokens junto al cerrar. Opcional: iaTokensBadgeId (para sincronizar el número desde tu JS), iaTokensTooltip (texto del data-tooltip).

openModal({
  overlayId: 'mi-modal-ia',
  title: 'Generar con IA',
  bodyHtml: '<p class="ubits-body-md-regular">Contenido.</p>',
  footerHtml: '<button type="button" class="ubits-button ubits-button--secondary ubits-button--md" onclick="closeModal(\'mi-modal-ia\')"><span>Cancelar</span></button><button type="button" class="ubits-button ubits-button--primary ubits-button--md" onclick="closeModal(\'mi-modal-ia\')"><span>Generar</span></button>',
  size: 'md',
  variant: 'ia',
  iaTokensRemaining: 42,
  iaTokensBadgeId: 'mi-modal-ia-tokens-badge',
  iaTokensTooltip: 'Número de tokens restantes.',
  closeOnOverlayClick: true
});

Variante promo (variant: 'promo')

Sin cabecera ni pie ni botón X. footerHtml se ignora. size solo admite xs o sm (cualquier otro equivale a sm). Cierre: tecla Escape, clic fuera si closeOnOverlayClick es verdadero, o botones en bodyHtml. Usa promoAriaLabel o title (no visible) para el aria-label del diálogo. En bodyHtml: .ubits-modal-promo-media (imagen) + .ubits-modal-promo-copy (texto y botón).

openModal({
  overlayId: 'promo-novedades',
  variant: 'promo',
  size: 'sm',
  title: 'Nuevas funciones con IA',
  promoAriaLabel: 'Nuevas funciones con IA',
  bodyHtml:
    '<div class="ubits-modal-promo-media"><img src="ruta/imagen.jpg" alt="" /></div>' +
    '<div class="ubits-modal-promo-copy">' +
    '<p class="ubits-body-lg-semibold ubits-modal-promo-title" id="promo-novedades-heading">Título</p>' +
    '<p class="ubits-body-sm-regular">Descripción breve.</p>' +
    '<button type="button" class="ubits-button ubits-button--primary ubits-button--md" onclick="closeModal(\'promo-novedades\')"><span>Explorar</span></button>' +
    '</div>',
  closeOnOverlayClick: true
});
