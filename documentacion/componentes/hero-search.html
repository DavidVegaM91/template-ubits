Hero search

Buscador principal del Inicio de Aprendizaje: título sobre un fondo con gradiente de marca y un campo pill con lupa. Sirve para invitar a buscar contenidos o competencias sin usar el Input genérico de formularios.

Preview del componente

El interruptor simula el borde reforzado cuando el usuario enfoca el buscador o hay búsqueda activa (clase ubits-hero-search__bar--active).

Barra activa (foco / búsqueda)

Características generales

Gradiente APP v3
Bloque hero con título centrado y altura mínima pensada para Inicio Aprendizaje.
Campo pill nativo
Input type search con botón lupa integrado; no sustituir por createInput en este patrón.
Textos configurables
Título, placeholder y etiquetas ARIA personalizables al montar el componente.
Lógica en la página
Modos exploración y resultados (debounce, skeleton, catálogo) los cablea la pantalla, p. ej. home-learn-search.js.

Prompts útiles

Copia el texto y adapta lo que va entre corchetes. Importa hero-search.css y hero-search.js antes del script de la página.

Insertar buscador hero en Inicio Aprendizaje

En [ubicación] de la pantalla Inicio Aprendizaje, monta el componente UBITS Hero search con mountHeroSearch: contenedor [id del contenedor], título [texto del título], placeholder [texto del placeholder]. Importa hero-search.css y hero-search.js. No uses createInput para el campo del hero.

Marcar barra activa al buscar

Cuando el usuario enfoca el buscador hero o escribe en él, llama setHeroSearchBarActive con el id [id de la barra] y true; al volver al estado idle con campo vacío, pásale false.
